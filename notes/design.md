# Design notes

The reasoning behind straitjacket's checks — why each rule exists, what it
costs, and where the lines are drawn — lives in the published docs at
[straitjacket.dev](https://straitjacket.dev), especially the explanation
pages (e.g. [slop-prose](https://straitjacket.dev/docs/explanation/slop-prose)).
This file is the working end: design questions that are still open, decisions
made but not yet written up properly, and sketches for rule families under
consideration (see the issue tracker for candidates like the
[note-hygiene family](https://github.com/zmaril/Straitjacket/issues/35)).

Fleet convention: every open source powderworks repo keeps its design
thinking in notes/design.md, so there is always one place to look.
